# 하네스 엔지니어링 (AI 에이전트 설계)

## 정의

모델 가중치가 아닌 에이전트를 구동하는 코드·프롬프트·도구 체계. Anthropic이 "하네스가 성능의 핵심"이라고 공식 언급.

## 사업적 의미

AI 경쟁력이 모델 크기에서 엔지니어링 설계로 이동. 하네스 IP가 핵심 전략 자산이 됐다.

## 실제 사례

Claude Code 50만줄 소스코드 유출 (하네스 구조 노출), 3계층 학습 (Model·Harness·Context), Workload-Harness Fit 프레임워크

## 관련 시그널

### AI 에이전트의 3계층 학습
*2026-04-07 · TLDR*

**So What**: 지속 학습 설계 시 모델 재학습 외 harness·context 계층 활용이 실용적 대안이 될 수 있다.

- Model 계층: 모델 가중치 자체를 업데이트하는 방식으로 가장 비용이 크다
- Harness 계층: 에이전트를 구동하는 코드, 프롬프트, 도구를 수정해 학습 효과를 낼 수 있다

### 클로드 코드 하네스 설계 유출
*2026-04-01 · 안될공학*

**So What**: 모델 가중치가 아닌 '하네스 설계'가 AI 경쟁력의 핵심이 된 시대, 배포 실수 하나가 전략적 IP를 노출할 수 있다.

- 유출 규모: 소스맵 포함 실수로 약 50만 줄 내부 타입스크립트 코드 복원 가능해짐
- 앤트로픽은 2026년 3월 24일 공식 글에서 '하네스 디자인이 에이전트 성능의 핵심'이라고 직접 명시

### AI 에이전트 워크로드 적합성
*2026-04-01 · TLDR*

**So What**: 모든 업무에 AI 에이전트가 맞는 건 아니며, 배포 전 적합성 판단 기준이 필요하다.

- 워크로드 특성 5가지: 볼륨, 가치, 검증 가능성, 시간 범위, 기타 차원
- 워크로드-하네스 적합성(Workload-Harness Fit) 이해가 프로덕션 배포의 핵심
