# Mnemo Wiki Schema

## 목적
전략 기획 및 투자 리서치를 위한 테마별 지식 컴파일 시스템

## 테마 문서 구조
각 테마는 wiki/ 폴더 내 하나의 .md 파일로 관리한다.

### 필수 섹션
1. **종합 판단** (문서 최상단)
   - 현재 시점에서의 핵심 판단을 3줄 이내로
   - 마지막 업데이트 날짜 명시
   - 확신도 표시: 🟢 높음 / 🟡 중간 / 🔴 낮음

2. **핵심 시그널 타임라인**
   - 시간역순 정렬
   - 각 항목에 출처 카드 ID, 날짜, 출처명 명시
   - 기존 판단을 강화하는 시그널: [+]
   - 기존 판단과 충돌하는 시그널: [⚠️]
   - 새로운 방향의 시그널: [NEW]

3. **열린 질문**
   - 이 테마에 대해 아직 답이 없는 전략적 질문
   - 새 카드가 질문에 답하면 시그널로 이동하고 질문에서 제거

4. **관련 테마**
   - 다른 wiki 문서로의 링크와 연결 맥락 한 줄 설명

## 컴파일 규칙
- 새 카드가 kept되면: 기존 테마 목록과 의미적 매칭 실행
- 매칭 점수 0.7 이상: 해당 테마 wiki에 시그널로 추가하고 종합 판단 재평가
- 매칭 테마 없음: 새 테마 생성 제안 (카드 3개 이상 누적 시 자동 생성)
- 기존 시그널과 모순 발견 시: [⚠️] 플래그 + 종합 판단에 불확실성 반영

## 린팅 규칙 (주 1회)
- 30일 이상 새 시그널 없는 테마: status → dormant으로 변경
- 열린 질문 중 답변된 것 정리
- 테마 간 새로운 연결 가능성 점검
- 종합 판단의 현행화 여부 점검
- 결과를 lint_report.md로 출력

## 아웃풋 생성 규칙
- 브리핑: 해당 기간 내 시그널 변화 요약 + 판단 변화 + 주의사항
- 투자 메모: 종합 판단 + 핵심 근거(시그널) + 리스크(충돌 시그널) + 열린 질문
- 비교 분석: 복수 테마/엔티티의 시그널을 병렬 배치하여 차이점 도출
